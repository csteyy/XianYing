# 飞书多维表格集成计划

## 整体数据流

采用与火山引擎 ASR 相同的**双通道**策略：iOS 直连飞书 API，Web 开发走 localhost 代理。

```
写入流程:
  iOS 真机 (isNative = true):
    App → Feishu: 获取 tenant_access_token
    App → Feishu: batch_create 写入100+条对话

  Web 开发 (isNative = false):
    App → Proxy: POST /feishu/write
    Proxy → Feishu: 转发请求

  Feishu → Bitable: AI字段捷径自动评测

读取流程:
  iOS 真机:
    App → Feishu: list records

  Web 开发:
    App → Proxy: POST /feishu/read
    Proxy → Feishu: 转发请求
```

## 多维表格 Schema

每条对话对应一行：

| 字段 | 类型 | 来源 |
|------|------|------|
| 对话文本 | 文本 | `TranscriptItem.text` |
| 发言者 | 文本 | `TranscriptItem.speaker` |
| 时间戳 | 数字 | `TranscriptItem.timestamp` |
| 序号 | 数字 | 自增 |
| 对谁发言 | AI 字段 | 飞书 AI 自动填 |
| 情绪基调 | AI 字段 | 飞书 AI 自动填 |
| 互动类型 | AI 字段 | 飞书 AI 自动填 |

## 服务层结构

```
src/services/feishu/
  config.ts      — 凭证与端点
  auth.ts        — tenant_access_token 获取与缓存
  bitable.ts     — batch_create 分片写入、list records 翻页读取
  client.ts      — isNative() 分流，iOS 直连 / Web 走代理
```

## 环境变量

```
VITE_FEISHU_APP_ID=cli_xxxxx
VITE_FEISHU_APP_SECRET=xxxxx
VITE_FEISHU_BITABLE_APP_TOKEN=xxxxx
VITE_FEISHU_BITABLE_TABLE_ID=tblxxxxx
```

## 关键 API

- **批量写入**：`POST /open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/records/batch_create`（500 条/批）
- **列出记录**：`GET /open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/records`
