# 后端 / 个人 Demo 上线计划

## 现状

- 框架：React + Vite + Capacitor ✅
- 音频采集、流式 ASR、文件 ASR、飞书集成、iOS WebSocket 插件、代理 ✅
- 缺失：API Keys、数据持久化、部分数据流断裂

## 代码需要补齐的部分

1. **storage.ts 持久化**：用 localStorage 存录音记录、设置、作品库
2. **DataAnalysisPage**：无飞书时，从 transcripts + speakers 生成分析
3. **Visualization3DPage**：接收真实的 participants/speeches
4. **HistoryPage / GalleryPage**：从 localStorage 加载真实记录和作品
5. **SettingsPage**：持久化到 localStorage
6. **PostRecordingEditPage**：飞书未配置时静默跳过

## 外部配置

### 火山引擎（必做）

1. 注册火山引擎并开通语音服务
2. 新建应用，获取 App Key、Access Key、Resource ID
3. 填入 `.env.local`

### 飞书（可选）

先不接，逻辑已做降级。

## iOS 真机部署步骤

```bash
cd ~/Desktop/XianYing/显影
npm run build
npx cap sync ios
npx cap open ios
```

在 Xcode 中：

1. Signing & Capabilities 勾选自动管理签名
2. 选择 Personal Team
3. Bundle Identifier 冲突时改为唯一值（如 `com.yourname.xianying`）
4. 连接 iPhone，选择设备，点击 Run

**注意**：免费签名 7 天过期，需重新 Run；开启开发者模式：设置 → 隐私与安全性 → 开发者模式。
